# Untitled Navigation And Photobook Design

## Goal

`무제 (Untitled)`를 더 단순하고 직관적인 온라인 잡지로 정리한다.  
기존의 레퍼런스 흔적과 과한 메뉴를 제거하고, 웹과 모바일 모두에서 편하게 읽히는 구조로 다듬는다.

## Scope

- 상단 메뉴를 `Home / Photo book / Submit / About`만 남기고 단순화
- 외부 레퍼런스 관련 문구와 복잡한 메뉴 구조 제거
- `Photo book`을 웹에서는 펼친 책처럼, 모바일에서는 한 장씩 스와이프로 넘기는 구조로 개편
- `Home`, `Submit`, `About`도 모바일 우선으로 단순화
- 전체 페이지의 이미지, 여백, 텍스트 폭을 재정리해 웹/모바일 모두 읽기 쉽게 개선

## Information Architecture

### Home

- 첫 화면은 큰 대표 이미지 1장과 짧은 소개 문장
- 주요 행동은 `Photo book 보기`
- 그 아래에는 사진 그리드와 최신 글 2~3개만 간결하게 배치
- 이전의 `Issues`, `Stories`, `Contact` 개별 진입은 제거

### Photo book

- 웹: 좌우 두 페이지가 펼쳐진 책 형태
- 모바일: 한 장씩 크게 보이는 단일 페이지 형태
- 모바일에서는 좌우 스와이프, 버튼 탭 모두 지원
- 페이지 번호와 캡션은 보조 정보로 간결하게 유지
- 읽기 흐름을 해치지 않도록 불필요한 컨트롤은 제거

### Submit

- 글 제출과 사진 제출만 유지
- 한 화면 안에서 전환 가능한 단순한 구조 사용
- 모바일에서는 세로 한 칸 폼으로 배치
- 이미지 업로드 구역은 크게 두고 설명은 짧게 유지

### About

- 서비스 설명을 짧게 유지
- 외부 레퍼런스 언급 제거
- 브랜드 설명은 “사진과 글로 채워가는 빈 책” 수준으로 정리

## Interaction Design

### Header

- 얇고 조용한 sticky header
- 데스크톱은 메뉴를 한 줄로 노출
- 모바일은 햄버거 또는 축약 메뉴로 전환
- 검색 입력과 과한 보조 UI는 제거

### Photo book navigation

- 데스크톱: 이전/다음 버튼과 썸네일 없는 단순 페이지 이동
- 모바일: 터치 스와이프 우선, 버튼은 보조 수단
- 현재 페이지 인디케이터는 상단 또는 하단에 작게 노출

## Visual Direction

- 전체적인 분위기는 조용하고 가벼운 에디토리얼 레이아웃
- 카드 수를 줄이고 페이지를 넓게 사용
- 큰 제목은 일부 화면에만 사용하고, 나머지는 한 단계 낮춘 크기 사용
- 이미지가 주인공이 되도록 텍스트 밀도를 낮춤
- 색상은 현재의 따뜻한 중성 톤을 유지하되 대비는 더 명확하게 조정

## Responsive Rules

- 모바일 우선으로 재배치
- 이미지에는 안정적인 `aspect-ratio` 적용
- 텍스트는 너무 긴 줄이 생기지 않도록 폭 제한
- 버튼과 터치 영역은 모바일에서 충분히 크게 유지
- 포토북 스와이프 영역은 화면 폭을 자연스럽게 채우도록 구성

## Implementation Notes

- `apps/web/src/main.js`에서 메뉴 구조와 뷰 렌더링 단순화
- `apps/web/src/styles.css`에서 레이아웃, 헤더, 포토북, 폼 전반 재구성
- `Photo book`에 포인터 기반 스와이프 로직 추가
- 기존 API 구조는 유지하고 UI 계층만 정리

## Risks And Mitigations

- 단순화 과정에서 기존 탐색 경로가 사라질 수 있다  
  -> 홈 안에서 사진과 최신 글 진입을 바로 제공한다

- 모바일 스와이프가 버튼 내비게이션과 충돌할 수 있다  
  -> 짧은 임계값 기반 스와이프 판정과 명확한 탭 타깃을 사용한다

- 한 화면에 정보가 너무 줄어들면 잡지 느낌이 약해질 수 있다  
  -> 홈 상단의 대표 이미지와 포토북의 펼침 레이아웃으로 에디토리얼 감성을 유지한다

## Success Criteria

- 상단 메뉴가 `Home / Photo book / Submit / About`만 보인다
- 홈이 더 단순해지고 모바일에서 첫 화면 읽기가 편하다
- 포토북이 모바일에서 자연스럽게 스와이프된다
- `API offline`, 경로 오류 없이 기존 데이터가 정상 노출된다
- 웹과 모바일 모두에서 텍스트와 이미지가 겹치지 않고 읽기 쉽다
